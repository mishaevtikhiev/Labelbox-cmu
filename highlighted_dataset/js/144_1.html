3.3. Reverse Polish Notation
You will now implement reverse polish notation - a method of converting infix to postfix notation. You can assume the input is in valid postfix form.

The reverse polish notation is the operation of converting the postfix expression into the operand order of another infix expression. 

For example, if $x$ is converted to $2x + 3$ and $y$ is converted to $x - y$, then $x y$ is converted to $x -y $. If the first operand (the variable or number) in the postfix expression contains no parentheses or other operators, then the operand order is unchanged. If the parentheses are used, then the postfix expression is evaluated first, then the operand order is applied.

The reverse polish notation algorithm relies on the evaluation method discussed in the previous section.

For the input $x y = 3 $, the reverse polish notation algorithm will output $3 x $ .

You can assume that the input is a well-formed postfix expression. It does not contain any parentheses.
